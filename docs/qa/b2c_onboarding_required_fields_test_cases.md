# B2C 입점 필수값 테스트 케이스 (CRM Dev1)

## 개요
- 대상: `온라인 예약 > 공비서로 예약받기 > 공비서 입점하기/수정하기`
- 샵: `샵 새성`
- 목적: 아래 3개 필수값 검증
  - 예약 상담 가능 번호
  - 소개글
  - 샵 소개 이미지

## 실행 환경
- URL: `https://crm-dev1.gongbiz.kr/signin`
- 계정: 환경변수 사용 권장 (`CRM_USER_ID`, `CRM_USER_PW`)
- 기준 일자: 2026-03-08

## 공통 절차
1. 로그인 후 대상 샵(`샵 새성`)으로 전환한다.
2. `온라인 예약 > 공비서로 예약받기`로 이동한다.
3. `공비서 입점하기` 또는 `온라인 예약 정보 > 수정하기` 진입한다.
4. 메인 저장 버튼(`data-track-id=b2c_info_save`) 활성 상태를 확인한다.

## 테스트 데이터
- 전화번호 정상값: `01012345678`
- 소개글 정상값: `샵 소개 테스트 문구입니다.`
- 이미지 정상값: PNG/JPG 1개 (20MB 이하)

## 비고
- 이미지는 파일 선택 후, 이미지 모달의 `저장`까지 완료되어야 메인 폼에 반영된다.
- 필수값 검증 관점에서 3개 항목은 모두 required로 동작한다.
