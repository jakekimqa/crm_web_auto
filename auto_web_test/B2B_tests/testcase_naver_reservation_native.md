# 네이버 예약 검증 테스트 케이스 (네이티브 앱)

> 웹 자동화 테스트(`test_naver_reservation.py`)와 동일한 시나리오를 네이티브 앱에서 구현하기 위한 상세 테스트 케이스

---

## 환경 정보

- **서버**: dev1 (crm-dev1.gongbiz.kr)
- **계정**: herrenail / gong2023@@
- **샵**: 네이버테스트 60분
- **담당자**: 공비서원장님, 구구직원, 아샌1, 김제크_직원계정
- **예약 날짜**: 2026-04-04 (토요일)

---

## 사전 조건

- 네이버 예약 4건이 4/4(토) 캘린더에 등록되어 있어야 함
- 각 담당자별 오후 2:00 (시나리오 4는 오후 3:00) 시간대에 예약이 존재해야 함
- 예약 고객명: **제크** (010-4158-4484)

## Phase 0: 로그인 + 샵 선택

### TC-0.1: 로그인 후 네이버테스트 60분 샵 진입

**상세 스텝:**
1. crm-dev1.gongbiz.kr/signin 접속
2. herrenail / gong2023@@ 로그인
3. 마이페이지 > 샵 목록에서 **"네이버테스트 60분"** 샵의 "샵으로 이동" 클릭

**검증:**
- 예약 캘린더 페이지(`/book/calendar`)로 이동 확인
- 좌측 상단 샵 이름: "네이버테스트 60분"

> **주의**: herrenail 계정에 샵이 많으므로, 스크롤하여 "네이버테스트 60분"을 찾아야 함

## Phase 1: 캘린더 이동 + 예약 수 확인

### TC-1.1: 4/4(토) 캘린더 일별 뷰 이동

**상세 스텝:**
1. 캘린더 뷰 > "일" 탭 선택
2. 날짜 네비게이션으로 2026-04-04 (토요일)로 이동

**검증:**
- 헤딩에 "26. 4. 4 (토요일)" 표시
- 캘린더에 **"제크"** 이름이 포함된 예약 블록이 **4개 이상** 존재

---

## Phase 2: 시나리오 1 — 공비서원장님 예약 검증

### TC-2.1: 예약 상세 — 쿠폰 정보 확인

**상세 스텝:**
1. 캘린더 일별 뷰에서 **공비서원장님** 컬럼의 **오후 2:00** "제크" 예약 클릭
2. 예약 상세 정보 페이지 진입 확인

**검증:**

| 항목 | 기대값 |
|------|--------|
| 담당자 | 공비서원장님 |
| 네이버 쿠폰 정보 | **5,000원 할인** 쿠폰 노출 |

---

### TC-2.2: 매출 등록 — 쿠폰 정보 연동 확인

**상세 스텝:**
1. 예약 상세 > "매출 등록" 버튼 클릭
2. 매출 등록 페이지 진입

**검증:**
- 매출 등록 페이지에 **"5,000원 할인"** 쿠폰 정보 표시

**복귀:** 캘린더로 돌아가기

---

## Phase 3: 시나리오 2 — 구구직원 예약 검증 (NPay 결제 요청 → 취소)

### TC-3.1: 예약 상세 — 쿠폰 정보 없음 확인

**상세 스텝:**
1. 4/4 캘린더 일별 뷰에서 **구구직원** 컬럼의 **오후 2:00** "제크" 예약 클릭
2. 예약 상세 정보 페이지 진입

**검증:**

| 항목 | 기대값 |
|------|--------|
| 담당자 | 구구직원 |
| 네이버 쿠폰 정보 | **없음** (섹션 자체가 노출되지 않아야 함) |

---

### TC-3.2: 매출 등록 — NPay 결제 요청 → 취소 플로우

**상세 스텝:**
1. "매출 등록" 버튼 클릭
2. 시술 메뉴 선택: **손 > 젤기본**
3. **"다음"** 버튼 클릭 → 결제 수단 선택 페이지로 이동
4. **"결제 요청"** 버튼 클릭
5. "네이버페이 매장 결제 요청" 모달 확인
6. **"결제 요청 전송"** 버튼 클릭
7. "고객 결제 진행 중" 모달 확인
8. **"결제 요청 취소"** 버튼 클릭

**검증:**

| 순서 | 검증 항목 |
|------|-----------|
| 3 | 결제 수단 선택 페이지로 정상 이동 |
| 5 | "네이버페이 매장 결제 요청" 모달 타이틀 노출 |
| 7 | "고객 결제 진행 중" 모달 타이틀 노출 |
| 8 | "결제 요청이 취소되었습니다" 토스트/메시지 노출 |

> **주의**: 취소 토스트 메시지가 빠르게 사라질 수 있음

**복귀:** 캘린더로 돌아가기 (변경 사항 미저장 alert 발생 시 "확인" 처리)

---

**상세 스텝:**

### TC-5.2: 매출 등록 — 시술 추가 + NPay 결제 요청 → 취소

**상세 스텝:**
1. "매출 등록" 버튼 클릭
2. 시술 메뉴 선택: **손 > 젤기본**
3. **"다음"** 버튼 클릭 → 결제 수단 선택 페이지로 이동
4. **"결제 요청"** 버튼 클릭
5. "네이버페이 매장 결제 요청" 모달 확인
6. 매장 결제 금액 확인
7. **"결제 요청 전송"** 버튼 클릭
8. "고객 결제 진행 중" 모달 확인
9. **"결제 요청 취소"** 버튼 클릭

**검증:**

| 순서 | 검증 항목 | 기대값 |
|------|-----------|--------|
| 3 | 결제 수단 선택 페이지 | 정상 이동 |
| 5 | 네이버페이 매장 결제 요청 모달 | 타이틀 노출 |
| 5 | 총 결제 금액 | **40,000원** |
| 5 | 예약금 차감 | **(네이버페이) -300원** |
| 6 | 매장 결제 금액 (input) | **39,700원** (40,000 - 300) |
| 8 | 고객 결제 진행 중 모달 | 타이틀 노출 |
| 9 | 결제 요청 취소 메시지 | "결제 요청이 취소되었습니다" |

> **주의**: 취소 토스트 메시지가 빠르게 사라질 수 있음

**복귀:** 캘린더로 돌아가기 (변경 사항 미저장 alert 발생 시 "확인" 처리)

## 참고: 예약 상세 정보 페이지 구조

```
예약 상세 정보
├── 고객 정보 (이름, 연락처, 방문/노쇼 횟수)
├── 예약 일시 (날짜, 시간)
├── 담당자
├── 시술 메뉴 / 총 금액
├── 시술 시간
├── 예약금 (네이버페이 금액 + 예약금 환불 버튼)
├── 네이버 쿠폰 정보 (있는 경우만 노출)
├── 예약 메모
└── [예약 수정] [매출 등록] 버튼
```

## 참고: NPay 결제 요청 플로우

```
매출 등록 페이지
└── "결제 요청" 버튼 클릭
    └── 네이버페이 매장 결제 요청 모달
        ├── 총 결제 금액
        ├── 예약금 차감 (네이버페이)
        ├── 매장 결제 금액 (input, 수정 가능)
        ├── 네이버 스타일 태그 (선택)
        └── "결제 요청 전송" 버튼
            └── 고객 결제 진행 중 모달
                └── "결제 요청 취소" 버튼
                    └── "결제 요청이 취소되었습니다" 토스트
```

## 참고: alert 처리

- 매출 등록 페이지에서 변경 사항이 있는 상태로 캘린더로 이동 시 **"변경 사항이 저장되지 않았습니다"** 브라우저 alert 발생
- 네이티브에서도 동일한 confirm dialog가 나올 수 있으므로 **"확인(OK)"** 처리 필요
