# B2B 시술사진 추가/삭제 자동화 테스트 프로세스

## 개요

매출등록 시 사진을 업로드한 뒤, 캘린더 예약 상세의 "최근 매출 메모"에서 사진이 정상적으로 저장·노출되는지,
그리고 매출 수정에서 사진을 삭제한 후 반영이 올바르게 되는지 검증하는 E2E 테스트.

---

## 1. 사전 조건 (셋업 완료 상태)

테스트 실행 전 아래 셋업이 완료되어 있어야 합니다.

| 항목 | 상태 |
|------|------|
| 자동화_1 고객 | 등록 완료 |
| 자동화_3 고객 | 등록 완료, 자동화_1과 패밀리 공유 |
| 예약 1 (자동화_1) | 오후 4:00 / 손 > 젤 기본 |
| 예약 3 (자동화_3) | 오후 6:00 / 손 > 케어 |
| 매출 1 (자동화_1) | 정액권 차감 20,000원 + **사진 10장** (0.5MB~4.9MB) |
| 매출 3 (자동화_3) | 현금 5,000 + 카드 5,000원 + **사진 10장** (0.5MB~4.9MB) |
| 매출 5 (자동화_3) | 패밀리 정액권 차감 + **사진 10장** (10MB x 10) |

---

## 2. 사진 업로드 프로세스 (매출등록 시)

매출등록 화면에서 사진 업로드는 다음 순서로 진행됩니다.

```
매출등록 페이지 진입
  → "사진" 탭 클릭
  → 사진 아이콘 (0/10) 클릭 → 사진 모달 열림
  → "사진 추가" 버튼 클릭 → file chooser 열림
  → 사진 파일 선택 (최대 10장 일괄)
  → 업로드 완료 대기
  → "저장" 버튼 클릭 → 사진 모달 닫힘
  → 매출 저장
```

### 업로드 사진 스펙

| 매출 | 대상 고객 | 사진 수 | 파일 크기 | 파일명 패턴 |
|------|----------|---------|----------|------------|
| 매출 1 | 자동화_1 | 10장 | 0.5MB ~ 4.9MB | test_photo_0_5MB.jpg ~ test_photo_4_9MB.jpg |
| 매출 3 | 자동화_3 | 10장 | 0.5MB ~ 4.9MB | test_photo_0_5MB.jpg ~ test_photo_4_9MB.jpg |
| 매출 5 | 자동화_3 | 10장 | 10MB x 10 | test_photo_10MB_1.jpg ~ test_photo_10MB_10.jpg |

---

## 3. 사진 확인 테스트

### 3-1. 자동화_1 사진 확인 (10장)

**프로세스:**

```
캘린더 일간 뷰
  → 자동화_1 예약 카드 클릭
  → 예약 상세 패널 노출
  → "최근 매출 메모" 버튼 클릭
  → #booking-item-total-history 패널 노출
  → img[alt^='최근 매출 메모 썸네일 이미지'] 개수 확인
```

**검증 항목:**

| 항목 | 기대값 |
|------|--------|
| 사진 개수 | 10장 |
| 깨진 이미지 수 | 0장 |

- 깨진 이미지 판정: `img.complete === false` 또는 `img.naturalWidth === 0`
- 이미지 로딩 대기: 모든 이미지가 `complete && naturalWidth > 0`이 될 때까지 최대 10초 polling

### 3-2. 자동화_3 사진 확인 (20장)

**프로세스:** 3-1과 동일 (대상 고객만 자동화_3으로 변경)

**검증 항목:**

| 항목 | 기대값 | 산출 근거 |
|------|--------|----------|
| 사진 개수 | 20장 | 매출 3에서 10장 + 매출 5에서 10장 |
| 깨진 이미지 수 | 0장 | — |

---

## 4. 사진 삭제 테스트 (자동화_3)

### 4-1. 삭제 프로세스

```
캘린더 일간 뷰
  → 자동화_3 예약 카드 클릭
  → 예약 상세 패널 노출
  → "매출 수정" 버튼 클릭
  → 매출 수정 페이지 진입
  → "사진" 탭 클릭
  → 사진 아이콘 (10/10) 클릭 → 사진 모달 열림
  → "이미지 삭제" 버튼을 5회 반복 클릭 (1번~5번 이미지 삭제)
  → 사진 모달 "저장" 버튼 클릭
  → dimmer 남아있으면 ESC로 닫기
  → "매출 저장" 버튼 클릭
  → alert "매출이 수정되었습니다" 확인 → accept
```

### 4-2. 삭제 상세 동작

| 단계 | 동작 | 비고 |
|------|------|------|
| 1 | 캘린더에서 예약 카드 클릭 | 일간 뷰, 스크롤하며 카드 탐색 |
| 2 | "매출 수정" 버튼 클릭 | 예약 상세 패널 내 버튼 |
| 3 | "사진" 탭 클릭 | `div.sc-f17959d8-1:has(h4:has-text('사진'))` |
| 4 | 사진 아이콘 클릭 | `div.sc-b56506f1-3:has(svg[icon='systemImage'])` → 사진 모달 열림 |
| 5 | 이미지 삭제 x 5회 | `button[aria-label*='이미지 삭제']` 첫 번째 버튼을 5회 반복 클릭 |
| 6 | 모달 저장 | `#modal-content button:has-text('저장')` 또는 `[role='dialog'] button:has-text('저장')` |
| 7 | 매출 저장 | `button:has-text('매출 저장')` 클릭 |
| 8 | alert 확인 | "매출이 수정되었습니다" 메시지 포함 여부 assert |

### 4-3. 삭제 후 검증

삭제 저장 완료 후 다시 캘린더로 돌아가 동일한 사진 확인 프로세스를 수행합니다.

**검증 항목:**

| 항목 | 기대값 | 산출 근거 |
|------|--------|----------|
| 사진 개수 | 15장 | 삭제 전 20장 - 삭제 5장 |
| 깨진 이미지 수 | 0장 | — |

---

## 5. 테스트 실행 순서

```
 1. test_setup_verify_photos_sales_1   — 자동화_1 사진 10장 확인
 2. test_setup_verify_photos_sales_3   — 자동화_3 사진 20장 확인
 3. test_setup_delete_photos_sales_3   — 자동화_3 사진 5장 삭제 → 15장 확인
```

> 별도 파일 `test_verify_photos.py`에도 사진 확인 테스트가 존재하며,
> 이 파일은 통계 테스트와 독립적으로 매출 1, 매출 3의 사진 10장씩만 검증합니다.
> (깨진 이미지 검증 없이 개수만 확인하는 간소화 버전)

---

## 6. 사진 확인 경로 요약

```
캘린더 (일간 뷰)
  └─ 예약 카드 클릭
       └─ 예약 상세 패널
            ├─ "최근 매출 메모" 클릭 → 사진 썸네일 확인 (조회)
            └─ "매출 수정" 클릭
                 └─ 사진 탭 → 사진 아이콘 클릭
                      └─ 사진 모달 (추가/삭제 가능)
                           ├─ "사진 추가" → file chooser → 업로드
                           └─ "이미지 삭제" → 개별 삭제 → 저장
```
