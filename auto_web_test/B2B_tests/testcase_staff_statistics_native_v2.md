### 테스트 데이터 규칙
- 고객명: `자동화_{MMDD}_N` (MMDD = 실행 날짜)
- 연락처: `010{MMDD}000N`
- 예: 3/28 실행 → 자동화_0328_1 / 01003280001

## 플랫폼별 제약 사항

| 항목 | Android Maestro | iOS Maestro | XCUITest / Espresso |
|------|----------------|-------------|---------------------|
| 한글 고객명 입력 | ❌ 불가 → 우회 필요 | ✅ | ✅ |
| 소개자 검색+선택 | ❌ 한글 검색 불가 | ⚠️ 드롭다운 선택 복잡 | ✅ |
| 드롭다운 옵션 선택 | ⚠️ | ⚠️ | ✅ |
| 금액 입력 (숫자) | ✅ | ✅ | ✅ |
| 통계 테이블 행-열 교차 검증 | ❌ 불가 | ❌ 불가 | ✅ (a11y id 필요) |
| 배치 대기 (5분) | ✅ repeat 가능 | ✅ repeat 가능 | ✅ |

### Android 한글 입력 우회 방안
1. **clipboard paste**: `adb shell input text` 또는 Maestro `runScript`로 클립보드 복사 후 붙여넣기
2. **사전 데이터**: 테스트 전 API/DB로 고객 데이터 미리 생성
3. **연락처 검색**: 고객 등록은 웹에서, 앱에서는 연락처(숫자)로 검색만

## Phase 1: 고객 추가

### TC-1.1: 고객 3명 등록

**데이터:**

| 순서 | 고객명 | 전화번호 | 소개자 |
|------|--------|----------|--------|
| 1 | 자동화_{MMDD}_1 | 010{MMDD}0001 | 없음 |
| 2 | 자동화_{MMDD}_2 | 010{MMDD}0002 | 자동화_{MMDD}_1 |
| 3 | 자동화_{MMDD}_3 | 010{MMDD}0003 | 자동화_{MMDD}_1 |

**Step 1: 고객 관리 진입**
- 동작: 하단 탭 "고객" 탭
- 대기: 고객차트 화면 표시

**Step 2: 고객 1 등록 (소개자 없음)**
- 동작: FAB 버튼 (우하단 + 버튼) 탭
- 동작: "직접 입력하기" 탭 (있는 경우)
- 대기: 고객 추가 폼 표시
- 동작: 고객 이름 필드 탭 → "자동화_{MMDD}_1" 입력
- 동작: 연락처 필드 탭 → "010{MMDD}0001" 입력
- 동작: "저장" 또는 "등록" 버튼 탭
- 대기: 고객 목록으로 복귀 또는 성공 표시

**Android 우회:**
```yaml
# 한글 입력 불가 시 — 연락처만 입력하고 이름은 clipboard paste
- runScript:
    script: |
      adb shell am broadcast -a clipper.set -e text "자동화_0328_1"
- longPressOn: "고객 이름"  # 붙여넣기 트리거
- tapOn: "붙여넣기"
```

**검증:**
- [MUST] 등록 성공 (에러 없이 폼 닫힘 또는 목록 복귀)

**Step 3: 고객 2 등록 (소개자: 고객 1)**
- 동작: FAB → 고객 추가 폼
- 동작: 이름 "자동화_{MMDD}_2" 입력
- 동작: 연락처 "010{MMDD}0002" 입력
- 동작: 소개자 필드 탭
- 대기: 소개자 검색 입력란 표시
- 동작: "자동화_{MMDD}_1" 입력 (또는 연락처 "010{MMDD}0001" 입력)
- 대기: 검색 결과 표시
- 동작: 검색 결과에서 "자동화_{MMDD}_1" 탭
- 동작: "저장" 탭

**검증:**
- [MUST] 등록 성공
- [SHOULD] 소개자 필드에 "자동화_{MMDD}_1" 표시

**Step 4: 고객 3 등록 (소개자: 고객 1)**
- Step 3과 동일, 이름/연락처만 변경

**Step 5: 중복 검증**
- 동작: 동일 이름+연락처로 재등록 시도
- [MUST] "이미 등록된 고객 연락처입니다" 텍스트 표시

## Phase 7: 직원 통계 검증

> **중요**: 매출 등록 완료 후 통계 배치 반영까지 최대 5분 소요
> 배치 대기 전략: 30초 간격으로 새로고침, 매출 건수 7건 확인될 때까지 (최대 12회)

### TC-7.0: 통계 페이지 진입 + 배치 대기

**Step 1: 통계 진입**
- 동작: 사이드 메뉴 또는 하단 탭 → "메뉴" → "통계" 탭
- 대기: 통계 화면 표시

**Step 2: 직원 통계 상세 진입**
- 동작: "직원 통계" 영역의 "자세히 보기" 탭
- 대기: 직원 통계 상세 화면 표시

**Step 3: 오늘 필터 적용**
- 동작: 기간 선택 영역 탭
- 동작: "오늘" 옵션 탭
- 동작: "기간 검색" 또는 "적용" 버튼 탭
- 대기: 데이터 로딩 완료

**Step 4: 배치 대기 (반복)**
- 조건: 화면에 "7건" 또는 매출 건수 7이 표시될 때까지
- 동작: 30초 대기 → 페이지 새로고침 (뒤로가기 → 다시 진입 또는 pull-to-refresh)
- 최대 반복: 12회 (총 6분)
- [MUST] 12회 내에 매출 건수 7건 확인

**Maestro 힌트:**
```yaml
- repeat:
    times: 12
    commands:
      - assertVisible:
          text: "7건"
          optional: true
      - runFlow:
          when:
            visible: "7건"
          commands:
            - stopRepeat
      - waitForAnimationToEnd
      - scroll  # pull-to-refresh 또는
      - back    # 뒤로가기 후 재진입
      - wait: 30000
```
