# B2B 통계 자동화 테스트 프로세스

## 개요

고객 추가 → 충전 → 패밀리 공유 → 예약 → 매출등록까지 셋업한 뒤,
직원 통계 / 고객 통계 / 채널별 매출 / 시간별·요일별 분석 / 고객 차트 필터를 검증하는 E2E 테스트.

---

## 1. 셋업 — 데이터 준비

### 1-1. 고객 추가 (3명)

| 순서 | 고객명 | 연락처 | 소개자 | 비고 |
|------|--------|--------|--------|------|
| 1 | 자동화_{mmdd}_1 | 010{mmdd}0001 | 없음 | 신규 일반 고객 |
| 2 | 자동화_{mmdd}_2 | 010{mmdd}0002 | 자동화_1 | 소개 고객 |
| 3 | 자동화_{mmdd}_3 | 010{mmdd}0003 | 자동화_1 | 소개 고객 |

- 고객차트 페이지에서 고객 등록 모달을 열어 이름/연락처 입력
- 2, 3번 고객은 소개자로 1번 고객을 선택
- 이미 존재하는 고객은 스킵
- 중복 연락처 등록 시도 → "이미 등록된 고객 연락처입니다" 모달 노출 확인

### 1-2. 정액권 충전 (자동화_1)

| 항목 | 값 |
|------|-----|
| 대상 고객 | 자동화_{mmdd}_1 |
| 충전 상품 | 20만원(현금) |
| 기대 실매출 증가 | +200,000원 |
| 기대 정액권 잔액 증가 | +220,000원 |

- 고객 상세 페이지 → 정액권 [충전] 버튼 클릭
- 드롭다운에서 "20만원(현금)" 선택 → 충전 완료
- 충전 전/후 좌측 요약 금액(실매출, 정액권 잔액)을 비교 검증

### 1-3. 패밀리 공유 (자동화_1 ↔ 자동화_3)

| 항목 | 값 |
|------|-----|
| 오너 | 자동화_{mmdd}_1 |
| 멤버 | 자동화_{mmdd}_3 |

- 자동화_1 고객 상세 → 패밀리 탭 → "패밀리 추가하기" 클릭
- 자동화_3 검색 → 선택 → 추가
- 패밀리 멤버 목록에 자동화_1, 자동화_3 모두 노출 확인

### 1-4. 티켓 충전 (자동화_2)

| 항목 | 값 |
|------|-----|
| 대상 고객 | 자동화_{mmdd}_2 |
| 충전 상품 | 10만원권(카드) |

- 고객 상세 페이지 → 티켓 [충전] 버튼 클릭
- "10만원권(카드)" 선택 → 충전 완료
- 충전 후 티켓 수량 증가 확인

### 1-5. 예약 등록 (3건)

| 예약 | 고객 | 시간 | 시술 카테고리 | 시술 메뉴 |
|------|------|------|-------------|----------|
| 예약 1 | 자동화_1 | 오후 4:00 | 손 | 젤 기본 |
| 예약 2 | 자동화_2 | 오후 5:00 | 티켓 | 10만원권 |
| 예약 3 | 자동화_3 | 오후 6:00 | 손 | 케어 |

- 캘린더 페이지 → 플로팅 버튼 → 예약 등록 모달
- 고객 검색/선택 → 시간 선택 → 시술 카테고리/메뉴 선택 → 등록

### 1-6. 캘린더 예약 확인

- 캘린더 일간 뷰에서 3건의 예약 카드 노출 확인
- 각 카드 클릭 → 상세 패널에서 고객명, 예약 시간 텍스트 검증

### 1-7. 매출등록 (5건)

#### 매출 1 — 자동화_1 (예약 기반, 정액권 차감)

| 항목 | 값 |
|------|-----|
| 진입 경로 | 캘린더 → 자동화_1 예약 카드 클릭 → 매출 등록 |
| 시술 금액 | 20,000원 (젤 기본) |
| 결제 수단 | 정액권 차감 20,000원 |
| 사진 업로드 | 10장 (0.5MB ~ 4.9MB) |
| 검증 | 정액권 초과 입력(21,000원) 시 alert 발생 확인 후 정상 금액(20,000원)으로 저장 |

#### 매출 2 — 자동화_2 (예약 기반, 티켓 차감)

| 항목 | 값 |
|------|-----|
| 진입 경로 | 캘린더 → 자동화_2 예약 카드 클릭 → 매출 등록 |
| 시술 금액 | 0원 (티켓 사용) |
| 결제 수단 | 티켓 차감 2매 (= 20,000원 상당) |
| 검증 | 티켓 차감 input 값이 "2"인지 확인 |

#### 매출 3 — 자동화_3 (예약 기반, 현금+카드)

| 항목 | 값 |
|------|-----|
| 진입 경로 | 캘린더 → 자동화_3 예약 카드 클릭 → 매출 등록 |
| 시술 금액 | 10,000원 (케어) |
| 결제 수단 | 현금 5,000원 + 카드 5,000원 |
| 사진 업로드 | 10장 (0.5MB ~ 4.9MB) |

#### 매출 4 — 미등록 고객 (비예약, 제품)

| 항목 | 값 |
|------|-----|
| 진입 경로 | 매출 메뉴 → 신규 매출등록 |
| 고객 유형 | 미등록 고객 |
| 상품 카테고리 | 제품 > 미분류 > 01_제품_테스트 |
| 결제 수단 | 네이버페이 5,000원 + 미수금 5,000원 |
| 실매출 | 10,000원 |

#### 매출 5 — 자동화_3 (비예약, 패밀리 정액권 차감)

| 항목 | 값 |
|------|-----|
| 진입 경로 | 매출 메뉴 → 신규 매출등록 |
| 고객 | 자동화_3 (고객 검색) |
| 시술 | 손 > 케어 |
| 결제 수단 | 자동화_1의 정액권으로 전액 차감 (패밀리 공유) |
| 사진 업로드 | 10장 (10MB x 10) |
| 검증 | 패밀리 공유 정액권 input이 노출되는지 확인 |

### 1-8. 매출 사진 검증

| 테스트 | 대상 고객 | 확인 내용 |
|--------|----------|----------|
| 사진 검증 1 | 자동화_1 | 캘린더 예약 상세 → 최근 매출 메모 → 사진 10장, 깨진 이미지 0장 |
| 사진 검증 3 | 자동화_3 | 캘린더 예약 상세 → 최근 매출 메모 → 사진 20장 (매출3 10장 + 매출5 10장), 깨진 이미지 0장 |
| 사진 삭제 | 자동화_3 | 매출 수정 → 사진 5장 삭제 → 저장 → 15장 확인, 깨진 이미지 0장 |

## 2. 통계 검증

> 통계 배치는 5분 주기로 반영됩니다. 테스트는 최대 10분(30초 x 20회) polling하며 배치 반영을 대기합니다.

### 2-1. 직원 통계 — 상품 유형별

통계 > 직원 통계 자세히 보기 > 오늘 필터 적용 > 상품 유형별 테이블

**기대값 (샵주테스트 직원 행)**

| 항목 | 기대값 | 산출 근거 |
|------|--------|----------|
| 매출 건수 | 7건 | 매출 1~5 (5건) + 정액권충전 + 티켓충전 |
| 실 매출 합계 | 320,000원 | 200,000(정액권) + 100,000(티켓) + 10,000(현금+카드) + 10,000(제품) |
| 시술 | 10,000원 | 매출3 현금+카드 |
| 정액권 판매 | 200,000원 | 정액권 충전 |
| 티켓 판매 | 100,000원 | 티켓 충전 |
| 예약 위약금 | 0원 | — |
| 제품 | 10,000원 | 매출4 미등록고객 제품 |
| 차감 합계 | 50,000원 | 정액권차감 20,000 + 티켓차감 20,000 + 패밀리정액권 10,000 |
| 정액권 차감 | 30,000원 | 매출1 20,000 + 매출5 10,000 |
| 티켓 차감 | 20,000원 | 매출2 |
| 총 합계 | 370,000원 | 실매출 320,000 + 차감 50,000 |

### 2-2. 직원 통계 — 고객 유형별 (실 매출 기준)

통계 > 직원 통계 > 고객 유형별 통계 탭 > 실 매출 기준 라디오 선택

**기대값 (샵주테스트 직원 행)**

| 신규 소개 | 2명 | 3건 | 110,000원 | 자동화_2: 티켓충전 100,000 / 자동화_3: 매출3 10,000 + 매출5 정액권차감(실매출 0) |

## 3. 고객 차트 필터 검증

### 3-1. 받은 시술 필터

| 필터 조건 | 기대 결과 |
|----------|----------|
| 손 > 젤 기본 | 자동화_1 노출 (매출1에서 젤 기본 시술) |
| 손 > 케어 | 자동화_3 노출 (매출3, 매출5에서 케어 시술) |

- 고객 > 고객차트 → 조건 추가 → "받은 시술" 선택
- 그룹 드롭다운: 손 / 시술 드롭다운: 젤 기본 → 결과 테이블에 자동화_1 확인
- 시술 드롭다운을 케어로 변경 → 결과 테이블에 자동화_3 확인

### 3-2. 담당자 필터

| 필터 조건 | 기대 결과 |
|----------|----------|
| 샵주테스트 | 자동화_1, 자동화_2, 자동화_3 전체 노출 |

- 고객 > 고객차트 → 조건 추가 → "담당자" 선택
- 담당자 드롭다운: 샵주테스트 → 결과 테이블에 3명 모두 노출 확인

## 4. 결과 전송

- 모든 테스트 완료 후 Slack webhook으로 각 테스트 항목의 PASS/FAIL 결과를 전송
- 전체 통과 시 "전체 통과!", 실패 항목 있을 시 "실패 항목 있음" 표시
